## 3. On-Page SEO Checklist

### 3.1 Page-Level Optimization

**URL Structure:**
- Use hyphens, not underscores: `/ac-service-delhi`
- Keep URLs short, descriptive, include primary keyword
- Maximum 3 levels deep: `/services/ac-repair/gurgaon`
- No stop words (and, or, but, the)
- Lowercase only
- Include target city for local pages

**Title Tag (60-70 characters):**
- Primary keyword at the beginning
- Brand name at the end, separated by pipe: `AC Service in Delhi | AC Repair Near You | Suraksha Home Care`
- Each page gets unique, non-duplicate title
- Include "Best", "Top", "Professional", "Affordable" sparingly
- Format for service pages: `[Primary Service] in [City] - [Secondary Benefit] | Suraksha Home Care`
- Format for location pages: `[Service] in [Locality], [City] - Best [Service] Near [Locality]`
- Format for blog posts: `[Keyword-Rich Title] - [Year] Guide | Suraksha Home Care`

**Meta Description (150-160 characters):**
- Include primary keyword + 1-2 secondary keywords naturally
- Clear value proposition and CTA
- Include phone number or "Book Now" prompt
- Differentiate from competitors
- Format: `Looking for [primary service] in [city]? Suraksha Home Care offers professional [service] at affordable prices. ✓ Trained staff ✓ Same day service ✓ Warranty. Call [phone] or book online!`

**H1 Tag (one per page):**
- Must include primary keyword
- Compelling, answers user intent
- Typically matches or is close to title tag
- Examples:
  - "Professional AC Service in Delhi – Same Day Repair & Maintenance"
  - "Best Plumber in Mumbai – 24/7 Emergency Plumbing Services"
  - "Home Deep Cleaning in Bangalore – Top Rated Cleaning Service"

**H2-H6 Headings:**
- H2s include secondary keywords
- Create a logical hierarchy
- One H2 should match the primary keyword variant
- Use FAQ-style H2s for informational content
- Maintain proper nesting (H2 → H3 → H4)

**Content Body:**

| Element | Requirement |
|---------|-------------|
| Word count | Service pages: 800-1500 words; Location pages: 600-1000 words; Blog posts: 1500-3000 words |
| Keyword density | 1-2% primary keyword, secondary keywords naturally placed |
| LSI keywords | Include 5-10 semantically related terms |
| Internal links | 3-5 links to related service/category pages |
| External links | 1-2 high-authority sources (where relevant) |
| Paragraph length | 3-4 sentences max |
| Bullet points | Use for lists, features, benefits |
| Bold text | Highlight key phrases (1-2 per 300 words) |
| Readability | Target Grade 8-10 level (for Indian English) |
| Multimedia | At least 1 image per 400 words, 1 video per 1500 words |

**Image Optimization:**
- Alt text: Descriptive, includes keyword naturally (not keyword stuffing)
- File naming: `ac-service-delhi-suraksha.jpg` (not `IMG_001.jpg`)
- Compression: Under 100KB for standard images, under 200KB for large images
- Format: JPEG for photos, PNG for graphics, WebP for modern browsers
- Dimensions: 1200x800px for hero images, 800x600px for content images
- Lazy loading enabled for all below-the-fold images

**Internal Linking Strategy:**
- Every service page links to 3-5 related services
- Service pages link to relevant location pages
- Blog posts link to relevant service pages
- Location pages link to primary service pages
- Use descriptive anchor text (not "click here")
- Implement pillar-cluster model:
  - Pillar: Primary service (e.g., AC Service)
  - Cluster: Location pages, service type pages (split AC service, window AC repair), blog posts
  - All cluster pages link back to pillar page

### 3.2 Content Quality Standards

- Original content (no AI-generated fluff, no duplicate content)
- Include specific data: pricing, service time, warranty period, technician qualifications
- Add expert quotes or tips from actual technicians
- Include step-by-step process descriptions
- Add before/after images where applicable
- Include FAQ section with schema markup
- Include trust signals: ratings, reviews, certifications, guarantee badges
- Add clear CTA buttons above the fold and at the end

## 4. Technical SEO Requirements

### 4.1 Core Web Vitals Optimization

**LCP (Largest Contentful Paint) — Target: <2.5 seconds:**
- Optimize hero image: Compress to WebP, max 70KB, responsive sizes
- Lazy load non-critical images and iframes
- Minify CSS, JavaScript, HTML
- Remove render-blocking resources
- Use critical CSS inline for above-the-fold content
- Preload hero image and primary font
- Reduce server response time (TTFB <200ms)

**FID (First Input Delay) / INP (Interaction to Next Paint) — Target: <100ms:**
- Defer non-critical JavaScript
- Remove unused JavaScript
- Minimize DOM size (target <1500 nodes)
- Use browser caching
- Optimize event handlers
- Break up long tasks (yield to main thread)

**CLS (Cumulative Layout Shift) — Target: <0.1:**
- Explicit width/height dimensions on all images and videos
- Reserve space for ads and embeds
- Avoid inserting dynamic content above existing content
- Use font-display: swap for web fonts
- Ensure all UI elements have reserved space
